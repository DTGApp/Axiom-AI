# Axiom — On-Device AI Assistant for Android

> *ChatGPT knows everything about the world but nothing about you.*
> *Axiom knows everything about your world.*

Axiom is a **privacy-first Android AI assistant** that runs entirely on-device — no internet, no account, no cloud. It learns your phone usage habits through on-device machine learning and gets smarter the longer you own the device.

**0 bytes sent. 0 servers crossed.** This is shown to the user — with actual milliseconds — after their very first inference.

## What Makes It Different

Every AI assistant on the market sends your queries to a remote server. Axiom physically cannot — there is no `INTERNET` permission in the manifest. The OS will block any network call, even accidental ones.

The model weight (~900 MB, Qwen2.5-1.5B Q4) lives on your device. The personal adapter that makes Axiom fast and personal (~50 KB) trains entirely from your own feedback. Nothing is ever uploaded. There is no backend to breach, no account to compromise, no privacy policy that changes.

The core differentiator is the **world model**: Axiom learns *your* habits and builds a private model of your world — your apps, your patterns, your rhythm. No other assistant on Android does this without a server.

## Feature Overview

### Home Screen

**Ask Axiom** — Voice or typed natural language. Three processing modes shown in real-time:
- 🔑 **Keyword** — instant regex match, no model needed
- 🧠 **Neural** — Qwen2.5-1.5B Q4 classified this (Vulkan GPU, ~200ms)
- ⚡ **Instant** — personal MLP adapter handled this (activates at 5+ interactions)

Confidence shown as percentage. Low-confidence results ask for confirmation before executing. Voice output via Android's built-in TTS.

**Context Card** — Live, personal insight at the top of the screen. Changes throughout the day as patterns are detected:
- *"You open Spotify every Tuesday evening — 6 of the last 7 weeks."*
- *"Battery at 18%. You typically charge around 9 PM — that's 2 hours away."*
- *"You've picked up your phone 47 times today, 31 opened Instagram."*
Every number is real, pulled from actual on-device usage data.

**Try Saying** — Rotating example card showing 3 contextual prompts (primary + 2 secondary). Changes every time the app opens. Tapping any example pre-fills the input. 10 rotating sets covering: app launching, navigation, media, calls, alarms, file search, social, and more.

**Learning Bar** — Persistent indicator of learning phase: Warming up → Early learning (< 5) → Improving (5–19) → Pattern forming (20–49) → Deeply trained (50+).

**My World Model** — Collapsible card showing what Axiom has learned. Expanded view shows a monospace habit breakdown: app name, hour, learned frequency. System apps filtered. Footer: *"All patterns learned on-device · 0 bytes sent anywhere."*

**On-Device Moment** — Shown once after the first successful inference. Displays actual inference milliseconds with *"0 bytes sent · 0 servers crossed."* Designed to be the screenshot moment.

**Dream Card** — Shows the dreaming animation during active learning cycles and a stats summary (interactions / trained / logs cleaned) after completion.

### Axiom Intelligence Screen

Accessed via 🔬 Intel pill or three-dots menu.

**Battery Doctor** — On-device AI diagnosis of battery health and background drain. Identifies the top drain app, estimates savings, shows screen-on time and pickup count. Falls back to template diagnosis if LLM is busy.

**Axiom Noticed** — Weekly pattern insight generated by `InsightEngine`. Plain English summary with supporting data points. System apps filtered. Minimum 6 days between new insights — can be forced via "Analyse Now."

**Auto Mode** — Current active profile (Sleep / Commute / Work / Focus / Gaming / Normal), recent switch log, and volume preference controls. Mode switches are always notifications with confirm/skip — never silently applied.

## Architecture

### Intelligence Stack

```
Tier 0   — Direct app launch / fallback          <1ms
Tier 0.3 — Contact + app action detection        <5ms    "call mom on WhatsApp"
Tier 0.5 — File search routing                   <5ms
Tier 0.6 — AI detector routing                   <5ms
Tier 0.7 — Smart actions                         <5ms    alarms, reminders, maps
Tier 1   — Keyword + adapter pipeline            <10ms
Tier 2   — LLM classification (ambiguous)        ~200ms  Qwen2.5-1.5B Q4, Vulkan
Tier 2.5 — Identity/creator questions            <1ms    hardcoded
Tier 3   — LLM conversational reply              ~200ms
```

**Personal MLP Adapter** — 12→64→101 MLP, SGD LR=0.01. Activates at 5+ interactions. Fully bypasses the LLM at 50+ interactions. Trains only from real accept/reject feedback — no synthetic data.

**Temporal Prediction Table** — `axiom_preds.csv` records intent × hour-of-day frequency. Updated daily by the usage scan. Enables time-aware predictions without any LLM.

### Always-On Service

`AxiomService` runs as a foreground service:
- `START_STICKY` — restarts if killed
- `BOOT_COMPLETED` — starts after reboot
- `stopWithTask=false` — survives task swipe on MIUI
- Background loops: sensor collection (5 min), proactive check (15 min), usage scan (daily)

### LLM Configuration

| Setting | Value |
|---------|-------|
| Model | Qwen2.5-1.5B Q4 |
| File | `axiom_seed_q4.gguf` (~900 MB) |
| GPU layers | All (Vulkan) |
| n_ctx classify | 150 tokens |
| n_ctx conversational | 512 tokens |
| Inference time | ~200ms Snapdragon mid-range |

## The Dream Cycle

Runs automatically when the phone is plugged in with ≥50% battery. No time restriction.

### Trigger Chain

```
POWER_CONNECTED broadcast
    └── ChargingReceiver.onReceive()
        └── startDreamIfEligible()  [battery ≥ 50%, 6h gap check]
            └── Intent(ACTION_START_DREAM) → AxiomService
                └── onStartCommand → maybeTriggerDream()
                    └── startDreamCycle()  [engineReady check]
```

If the engine isn't loaded yet, `startDreamCycle` schedules a 90-second retry. The 6-hour deduplication timestamp is written only **after** all guards pass — a failed attempt never blocks the next one.

### What Happens During a Dream

1. SharedPrefs `dream_state = "dreaming"` written
2. `ACTION_DREAM_STARTED` broadcast → animation starts on home screen
3. Ongoing notification: *"Axiom is dreaming..."*
4. `AxiomEngine.runDreamLearning()` runs on background thread
5. `axiom_events.jsonl` compacted: keeps last 48h + all surprisal events, capped at 500 lines
6. `dream_summary.json` written: events_before, events_after, removed, surprisal_events, adapter stats
7. SharedPrefs `dream_state = "complete"`, result stored
8. `ACTION_DREAM_COMPLETE` broadcast → home screen shows summary stats
9. Notification updated

## The Dreaming Animation

`DreamingAnimationView` — custom Canvas View, no OpenGL, no third-party library.

**Visual:** 14 cyan nodes (`#00E5FF`) drifting on a dark background (`#080B14`). Nearby nodes (within 220px) draw glowing edges that fade with distance. Each node pulses independently — brightness and radius oscillate on a sine wave at its own phase. Outer glow rendered as a semi-transparent larger circle behind each node.

**Performance:** 25fps (`FRAME_MS = 40L`). Automatically pauses when view is detached from window. Battery-friendly.

**When it shows:**
- Home screen → Dream Card becomes visible → animation starts
- Triggered live by `ACTION_DREAM_STARTED` broadcast if app is open
- Triggered by `onResume → updateDreamCard()` if app was closed during dreaming (reads SharedPrefs state)
- Stops on `ACTION_DREAM_COMPLETE`, card switches to stats view

## System Health Check

Open via three-dots menu → ⚕ System Health. Tap Refresh to re-run all checks.

Reads on-device files and SharedPreferences directly — no service binding required.

**18 checks across 5 sections:**

| Section | Checks |
|---------|--------|
| Core Engine | LLM model file · AxiomEngine JNI · Native library |
| Learning Pipeline | Events log · MLP Adapter · Predictions table · Usage scan |
| Dream Cycle | Dream state · Last dream ran · Dream summary · Dream data file |
| Intelligence | Weekly insight · Auto Mode · Life Context |
| App State | Onboarding · Milestones · Install age · Free storage |

**Dot colours:** 🟢 Green = healthy · 🟡 Amber = stale / not yet initialised · 🔴 Red = missing or broken · ⚫ Grey = not applicable yet

**Staleness thresholds:** Events 3d · Adapter 7d · Usage scan 2d · Dream 3d · Insight 14d · Storage amber <500 MB, red <200 MB

## MIUI / Xiaomi Notes

**App launching:** MIUI limits `getLaunchIntentForPackage` to ~18 apps. Axiom uses explicit `ComponentName` intents from the launcher cache, with package-only fallback.

**SpeechRecognizer double-fire:** MIUI fires `onResults` twice per utterance. Debounced on time (5 seconds) rather than text content.

**Battery optimisation:** Users must manually add Axiom to the battery exemption list:
`Settings → Battery → App Battery Saver → Axiom → No restrictions`

**`stopWithTask=false`:** Declared in manifest for `AxiomService`. Required to survive task swipe on MIUI.

## Known Limitations

**PACKAGE_USAGE_STATS onboarding** — Biggest new-user friction point. Cannot be granted via runtime dialog. Needs a deep-link to the exact Axiom entry in Usage Access settings with a visual walkthrough. Without this, many users will never have the learning pipeline active.

**Model download UX** — No resume on failure, no download ETA. Fix: byte-range resume + MB/s display.

**LLM concurrency** — Three code paths (`AxiomService.infer`, `BatteryDoctor`, `InsightEngine`) each have independent `llmBusy` flags. Simultaneous calls cause a native crash. Fix: single mutex in `AxiomEngine`.

**Context length guard** — Long conversational prompts silently truncate at n_ctx=512. No user warning. Fix: pre-measure token count before JNI call.

**Exact alarm drift** — Dream alarm uses `setAndAllowWhileIdle` (≤15 min drift) rather than exact alarms, which require explicit permission on Android 12+. Can drift to 30+ min during deep Doze. Acceptable for overnight dreaming.

## Privacy

- **Zero network permissions** — `INTERNET` absent from manifest. OS enforces this.
- **All weights on-device** — copied from assets on first launch, no download required after.
- **All learning data in `filesDir`** — inaccessible to other apps without root.
- **Adapter contains no raw data** — only abstract weight values. Queries are never stored in it.
- **No analytics, no crash reporting, no telemetry of any kind.**
- **No account, no login, no server.**
